# Implementation Plan: Improve Luma CLI Artifact Archiving

This plan addresses the issue where Luma CLI's Step 5 (`action_archive_artifacts`) only moves local markdown files from the project root but fails to automatically sync artifacts generated by Antigravity IDE and Gemini CLI from their respective hidden directories.

## Current State & Problem
- **`luma_core/actions.py`**: The `action_archive_artifacts` function successfully creates the target `docs/features/<issue>` directory and moves local files (`plan.md`, `spec.md`, etc.). However, it only prints a message saying AI Brain artifacts are handled by `ai_brain_sync.py` but never actually calls the sync functions.
- **`luma_core/ai_brain_sync.py`**: Contains `AntigravityBrain.sync_to_repo` and `GeminiCLIBrain.sync_to_repo` which correctly locate the latest session in `~/.gemini/` and copy the artifacts to `docs/features/<issue>/ai_brain`. These functions are currently unutilized in the standard workflow.

## Proposed Changes

We will modify `action_archive_artifacts` to actively call the sync functions from `ai_brain_sync.py` after processing local files.

### `luma_core/actions.py`

#### [MODIFY] luma_core/actions.py
1. **Import Sync Classes**: Import `AntigravityBrain` and `GeminiCLIBrain` from `luma_core.ai_brain_sync`.
2. **Execute Antigravity Sync**: Call `AntigravityBrain.sync_to_repo(project["path"], combined_number)`.
3. **Execute Gemini CLI Sync**: Call `GeminiCLIBrain.sync_to_repo(project["path"], combined_number)`.
4. **Update Console Output**: Log the results of these sync operations (e.g., "✅ Synced 3 Antigravity artifacts", "✅ Synced 1 Gemini CLI artifact") alongside the local file archiving results to provide clear feedback to the user.

## Verification Plan

1. **Manual Verification**:
   - Run the Luma CLI flow for an active issue.

   - Check the `docs/features/<issue>/ai_brain` directory to confirm that `task.md`, `walkthrough.md`, `implementation_plan.md`, and `gemini_chat_log.md` are correctly copied from `~/.gemini/`.
