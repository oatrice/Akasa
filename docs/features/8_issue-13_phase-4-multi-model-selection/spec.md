## 1. เป้าหมาย (The 'Why')

เพื่อเพิ่มความยืดหยุ่นและอำนาจในการควบคุมให้กับผู้ใช้ ทำให้พวกเขาสามารถเลือกใช้ LLM ที่เหมาะสมกับงานหรือความชอบส่วนตัวได้โดยตรงผ่านคำสั่งในแชท การมีหลายโมเดลให้เลือก (เช่น Claude สำหรับการเขียนโค้ด, Gemini สำหรับความเร็ว) จะช่วยยกระดับ Akasa จากบอทที่มีความสามารถเดียวไปสู่เครื่องมืออเนกประสงค์

## 2. ผู้ใช้งาน (User Persona)

| Persona | ลักษณะ |
|---|---|
| **End User** | ผู้ใช้งานที่ต้องการปรับแต่งประสบการณ์การใช้งานและเลือกเครื่องมือ (LLM) ที่ดีที่สุดสำหรับคำถามแต่ละประเภท |
| **The System (Akasa Backend)** | ระบบเบื้องหลังที่ต้องจัดการสถานะ (state) การเลือกโมเดลของผู้ใช้แต่ละคนและนำไปใช้งานอย่างถูกต้อง |

## 3. เส้นทางของผู้ใช้ (User Journey)

**ในฐานะ**ผู้ใช้งาน
**ฉันต้องการ** พิมพ์คำสั่ง `/model <model_name>` ในแชท
**เพื่อที่ฉันจะ** สามารถเปลี่ยน AI ที่จะมาตอบคำถามของฉันในการสนทนาครั้งถัดๆ ไปได้

## 4. เกณฑ์การยอมรับ (Acceptance Criteria)

- [ ] ระบบต้องสามารถประมวลผลคำสั่งที่ขึ้นต้นด้วย `/model` ได้
- [ ] เมื่อผู้ใช้ส่ง `/model <alias>` (เช่น `/model claude`), ระบบจะต้องตรวจสอบว่า `alias` นั้นถูกต้องหรือไม่
- [ ] หาก `alias` ถูกต้อง, ระบบจะต้องบันทึกการตั้งค่าโมเดลสำหรับผู้ใช้คนนั้น (ตาม `chat_id`) และส่งข้อความยืนยันกลับไป
- [ ] หาก `alias` ไม่ถูกต้อง, ระบบจะต้องส่งข้อความแสดงข้อผิดพลาดพร้อมรายชื่อ `alias` ทั้งหมดที่สามารถใช้ได้
- [ ] ในการสนทนาครั้งถัดไป, ระบบจะต้องใช้โมเดลที่ผู้ใช้ได้ตั้งค่าไว้
- [ ] การตั้งค่าโมเดลของผู้ใช้จะต้องถูกจดจำข้ามเซสชัน (persist)
- [ ] หากผู้ใช้ยังไม่เคยตั้งค่าโมเดล, ระบบจะใช้โมเดลเริ่มต้น (default)
- [ ] ผู้ใช้สามารถส่ง `/model` (ไม่มี argument) เพื่อตรวจสอบโมเดลที่ใช้งานอยู่ปัจจุบันและรายการโมเดลทั้งหมดที่เลือกได้

## 5. กรณีตัวอย่าง (Specification by Example - SBE)

### **Scenario 1: ผู้ใช้เปลี่ยนโมเดลสำเร็จ (Happy Path)**

**GIVEN** ผู้ใช้กำลังใช้โมเดลเริ่มต้น (e.g., `gemini`)
**WHEN** ผู้ใช้ส่งคำสั่ง `/model claude`
**THEN** ระบบจะบันทึกว่าผู้ใช้คนนี้ต้องการใช้โมเดล `claude`
**AND** บอทจะตอบกลับด้วยข้อความยืนยัน เช่น "✅ Model updated to Claude."
**AND** เมื่อผู้ใช้ส่งคำถามถัดไป, ระบบจะใช้โมเดล `claude` ในการประมวลผล

**ตัวอย่าง:**

| User Action | System Behavior | Bot's Reply |
|---|---|---|
| `/model claude` | 1. Validate `claude` alias.<br>2. Save preference for user's `chat_id`. | "✅ Model selection updated to: Claude 3.5 Sonnet." |
| "What is an AST?" | 1. Get user's preference (`claude`).<br>2. Call LLM Service with Claude model. | (An answer generated by Claude) |

---

### **Scenario 2: ผู้ใช้เลือกโมเดลที่ไม่มีอยู่จริง**

**GIVEN** ผู้ใช้ต้องการเปลี่ยนโมเดล
**WHEN** ผู้ใช้ส่งคำสั่ง `/model gpt5` ซึ่งเป็น alias ที่ไม่ถูกต้อง
**THEN** ระบบจะไม่เปลี่ยนแปลงการตั้งค่าใดๆ
**AND** บอทจะตอบกลับด้วยข้อความแสดงข้อผิดพลาดและรายการโมเดลที่ถูกต้อง

**ตัวอย่าง:**

| User Action | System Behavior | Bot's Reply |
|---|---|---|
| `/model gpt5` | 1. Validate `gpt5` alias.<br>2. Find no match. | "❌ Invalid model 'gpt5'.<br>Available models:<br>- `claude`<br>- `gemini`<br>- `gpt4`" |

### **Scenario 3: ผู้ใช้ตรวจสอบสถานะโมเดลปัจจุบัน**

**GIVEN** ผู้ใช้ได้ตั้งค่าโมเดลเป็น `claude` ไว้ก่อนหน้านี้
**WHEN** ผู้ใช้ส่งคำสั่ง `/model` (โดยไม่มี argument)
**THEN** ระบบจะดึงการตั้งค่าปัจจุบันของผู้ใช้
**AND** บอทจะตอบกลับพร้อมแจ้งสถานะปัจจุบันและรายการตัวเลือกทั้งหมด

**ตัวอย่าง:**

| User Action | System Behavior | Bot's Reply |
|---|---|---|
| `/model` | 1. Get user's preference from storage.<br>2. Find that it is set to `claude`. | "❇️ Current model: `Claude 3.5 Sonnet`<br><br>To switch, use `/model <alias>`:<br>- `claude`<br>- `gemini`<br>- `gpt4`" |

## 6. สิ่งที่ไม่ได้ทำใน Scope นี้ (Out of Scope)

- การเพิ่มโมเดลใหม่ๆ เข้าไปในรายการตัวเลือกแบบไดนามิก
- การจำกัดสิทธิ์ผู้ใช้ในการเลือกโมเดล (ทุกคนสามารถเลือกได้)
- การแสดงค่าใช้จ่ายหรือ token usage ของแต่ละโมเดล
