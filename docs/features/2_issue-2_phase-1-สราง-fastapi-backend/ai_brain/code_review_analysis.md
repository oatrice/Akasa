## 🧪 Test Suggestions (ข้อเสนอแนะสำหรับการเขียน Test เพิ่มเติม)

ตัว Reviewer ให้คำแนะนำเพิ่มเติมในการพัฒนา Test Suite เพื่อให้แข็งแรงขึ้นอีก 3 กรณี:

1. **Dependency/Database Failure Test (ทดสอบกรณีที่ระบบหลังบ้านล่ม):**
   - **แนวคิด:** ตอนนี้ `/health` แจ้งว่า `200 OK` เสมอ แต่ในชีวิตจริงถ้าระบบฐานข้อมูลล่ม ควรจะแสดง `503 Service Unavailable`
   - **สถานะ:** ยังไม่ต้องทำใน Phase นี้ เพราะ **Phase 1 เป็นแค่ Foundation ยังไม่มี Database** การเพิ่ม test แบบนี้เข้ามาตอนนี้รังแต่จะทำให้จำลอง (mock) ลำบากและนอกสโคป

2. **URL with Trailing Slash (ทดสอบกรณีพิมพ์ / ตอนจบ):**
   - **แนวคิด:** เพื่อป้องกันความสับสนเกี่ยวกับการพิมพ์ `GET /health/` แทนที่จะเป็น `/health` ว่าระบบควรจัดการอย่างไร ส่วนมากแนะนำให้ตอบกลับ `404 Not Found` กลับมาหากเราเข้มงวดกับเส้นทาง
   - **สถานะ:** ทำได้ง่าย รวดเร็ว และเสริมเรื่องความแม่นยำของ Route ได้ดี

3. **API Schema Validation (ทดสอบการสร้างเอกสารอัตโนมัติ):**
   - **แนวคิด:** ทดสอบด้วยการดึงไฟล์จาก `/openapi.json` เพื่อยืนยันว่า Endpoint ของ `/health` ถูกแท็ก (Tag) ว่าเป็น "Monitoring" ถูกต้องตามที่ระบุไว้ใน `app/routers/health.py` และไม่มีข้อมูล sensitive รั่วไหล
   - **สถานะ:** ทำได้ เป็นเรื่องที่ควรรักษามาตรฐานให้ดีตั้งแต่ต้น
